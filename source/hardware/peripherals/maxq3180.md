# MAXQ3180 — polyphase energy-measurement AFE

The MAXQ3180 is a low-power, multifunction, three-phase energy-measurement AFE
with an on-chip MAXQ20 core, DSP, and a **slave-only SPI** port. On the iPDU it
measures per-phase / per-load-segment voltage, current, active/reactive/apparent
power, energy, power factor and line frequency; the NS9360 is the SPI master and
reads results over DMA. The 8 MHz crystal Y4 is its clock; the KSZT770 CT (PT1)
supplies current sense. [`ANALYSIS.md`](https://github.com/mithro/ai-shenanigans-for-bmcs/blob/main/hpe-ipdu-firmware/ANALYSIS.md) [MAXQ3180 DS p.15](#sources)
Vendor confirms "an SPI port is provided for interfacing with a host
controller"; the device has 68 configuration registers, of which ~10 must be
initialised for basic operation. [MAXQ3180 AN4642](#sources)

Electrical/format constraints: [MAXQ3180 DS p.12,19-21](#sources)

- Max SPI clock = **fSYS/4** (2 MHz at fSYS = 8 MHz). Full-duplex; 8-bit
  characters only (no 16-bit), **LSB byte first** for multi-byte registers.
- Default format is CKPOL = 0, CKPHA = 0 (SPI mode 0): clock idles low, data
  sampled on the rising edge. Configurable via `R_SPICF` (0x052).
- Inter-byte spacing must be **≥ 400 system clocks** so the slave can process
  each byte; a stalled transfer aborts after `COM_TIMO` (default 320 ms).
- Optional per-transaction **CRC** (enable via OPMODE1.CRCEN): polynomial
  $P = x^8 + x^5 + x^4 + 1$, over all bytes except the sync bytes. Vendor strongly
  recommends enabling CRC for reliable comms. [MAXQ3180 DS p.23](#sources)

## SPI command format [MAXQ3180 DS p.22-23](#sources) (Tables 1, 2)

Every transaction starts with two command bytes. Byte 1 carries the command
code and data length; byte 2 carries the register address. On MISO the slave
echoes fixed sync bytes 0xC1 (during cmd byte 1) and 0xC2 (during cmd byte 2),
then an ACK (0x41) / NACK (0x4E) handshake.

```{list-table} Command byte 1 encoding
:header-rows: 1
:widths: 20 30 50

* - Field
  - Bits
  - Values
* - Command code
  - 7:6
  - 00 = Read, 10 = Write (01/11 reserved)
* - Data length
  - 5:4
  - 00 = 1 B, 01 = 2 B, 10 = 4 B, 11 = 8 B
* - Address MSB
  - 3:0
  - High nibble of the 12-bit register address
```

```{list-table} Transaction sequence (read vs. write)
:header-rows: 1
:widths: 18 41 41

* - Step
  - Read
  - Write
* - Cmd byte 1
  - master: `00`+len+addrMSB / slave: 0xC1
  - master: `10`+len+addrMSB / slave: 0xC1
* - Cmd byte 2
  - master: addrLSB / slave: 0xC2
  - master: addrLSB / slave: 0xC2
* - Data bytes
  - master: dummy / slave: data (LSB→MSB)
  - master: data (LSB→MSB) / slave: ACK
* - CRC (optional)
  - slave sends CRC
  - master sends CRC
* - Sync
  - slave: ACK (0x41) when ready, else NACK (0x4E)
  - slave: ACK (0x41) when processing complete
```

To read the calculated **virtual power registers**, the host first checks
`IRQ_FLAG.DSPRDY` (bit 14) = 1, reads them within 50 % of the DSP cycle time,
then clears DSPRDY itself (the device only sets it). [MAXQ3180 DS p.24](#sources)

## Key control & status registers

**STATUS (0x000)** — global status [MAXQ3180 DS p.28](#sources)

```{list-table} STATUS register
:header-rows: 1
:widths: 12 16 72

* - Bit
  - Name
  - Meaning
* - 7,3
  - —
  - Reserved
* - 6
  - CROFF
  - HF crystal failed; running on internal ring osc (energy accumulation inaccurate)
* - 5
  - PORF
  - Last reset was power-on reset (host clears)
* - 4
  - WDTR
  - Last reset was watchdog (host clears)
* - 2
  - PHSEQ
  - Voltage phase sequence: 0 = A-B-C, 1 = A-C-B
* - 1
  - REVCFQ
  - CFQ pulse output quantity is reverse (negative)
* - 0
  - REVCFP
  - CFP pulse output quantity is reverse (negative)
```

**OPMODE0 (0x001)** [MAXQ3180 DS p.29](#sources): bit 4 EXTCLK (use XTAL1 as clock
input), bit 3 SWRES (soft reset, self-clearing), bit 2 DSPDIS (disable DSP
routines), bit 1 LOWPM (switch to 1 MHz internal ring oscillator).

**OPMODE1 (0x002)** [MAXQ3180 DS p.29-30](#sources): bit 3 CRCEN (append/expect SPI CRC
byte), bit 2 POPOL (pulse-output polarity), bits 1:0 CONCFG (per-phase power
connection: 00 wye/isolated, 01 four-wire delta, 10 two-voltage wye, 11
three-wire delta).

**OPMODE2 (0x003)** [MAXQ3180 DS p.31-32](#sources): bit 6 DHARA / bit 5 DFUNA (disable
auto filter-coefficient determination), bit 4 DFUN (disable fundamental-mode
calc), bit 3 LINFRM (current-offset calibration method), bits 2:1 WIRSYS
(apparent-power wiring: 1P3W/3P4W, 3P3W, 3V3A), bit 0 APPSEL ($S = V_\text{rms} \cdot I_\text{rms}$ vs
$S = \sqrt{P^2 + Q^2}$).

**IRQ_FLAG (0x004) / IRQ_MASK (0x006)** — 16-bit [MAXQ3180 DS p.33-34](#sources). Each
IRQ_MASK `Exxx` bit gates the matching IRQ_FLAG bit onto the IRQ pin.

## Measurement units, line frequency, power factor

- Results are stored in internal "meter units" and converted to real units via
  the user-set coefficients **VOLT_CC / AMP_CC / PWR_CC / ENR_CC**.
  [MAXQ3180 DS p.64](#sources) (Tables 5, 6): $\text{MU_VOLT} = V_\text{FS}/2^{24}$, $\text{MU_AMP} = I_\text{FS}/2^{24}$,
  $\text{MU_PWR} = V_\text{FS} \cdot I_\text{FS}/2^{32}$, $\text{MU_ENR} = V_\text{FS} \cdot I_\text{FS} \cdot t_\text{FR}/2^{16}$.
- **LINEFR (0x062)** — line frequency, 16-bit, **1 LSB = 0.001 Hz**.
  [MAXQ3180 DS p.44](#sources)
- **PF.T (0x867)** and per-phase X.PF (A 0x1C6 / B 0x2B2 / C 0x39E) — power
  factor in units of 0.00001 (unity = decimal 100000 = 0x0186A0), two's
  complement. [MAXQ3180 DS p.53](#sources)
- Phase-angle registers (VBPH/VCPH) are in units of 0.01°. [MAXQ3180 DS p.53](#sources)

## What the iPDU firmware does with the MAXQ3180

From reverse engineering of the NET+OS firmware [`ANALYSIS.md`](https://github.com/mithro/ai-shenanigans-for-bmcs/blob/main/hpe-ipdu-firmware/ANALYSIS.md):

- SPI transfers are **DMA-based**; four error strings exist —
  `spi tx/rx DMA Cache error` and `spi slave tx/rx DMA Cache error` — implying
  the NS9360 also runs an SPI slave path (to extension bars). SPI-B is disabled
  by GPIO mux (`GPIO Config #1 = 0x33333333`); **Serial Port C is the strongest
  MAXQ3180 SPI candidate** (Control-A refs, no Bit-Rate refs).
- Calibration CLI (via J25): `gmcal` (calibrate Maxim voltage), `cc` (calibrate
  Maxim current, p|c), `gmsave` (persist to flash), `gmstats` / `gmstats2`
  (read back / secondary PDU), `mgain` (gain values). Status strings track a
  voltage → current → verify sequence; the host must reload calibration
  constants after every reset. [`ANALYSIS.md`](https://github.com/mithro/ai-shenanigans-for-bmcs/blob/main/hpe-ipdu-firmware/ANALYSIS.md) [MAXQ3180 DS p.65](#sources)
- Reported quantities surface in RIBCL XML / web UI as `<TOTAL_WATTS>`,
  `<TOTAL_VA>`, `<VOLTAGE>`, `<ENERGY>`, per-segment current and power factor
  (`Core%dr%dVA`, `Core%dr%dPF`), and system frequency. [`ANALYSIS.md`](https://github.com/mithro/ai-shenanigans-for-bmcs/blob/main/hpe-ipdu-firmware/ANALYSIS.md)
