# MAXQ3180 — energy-metering AFE

A Maxim polyphase energy-measurement analog front-end used by the HPE iPDU to
meter the power it distributes. The NS9360 talks to it over a serial link; the
stock NET+OS firmware polls it for voltage, current, power, and energy figures
that surface in the PDU's web UI.

```{list-table}
:header-rows: 0
:widths: 30 70

* - Interface
  - serial (SPI-class) command/response to the NS9360
* - Measures
  - per-phase V/I, active/reactive power, energy accumulation
```

## Relevance

- **Emulation** — a QEMU model needs to answer the firmware's metering polls
  with plausible readings so the PDU application initialises and serves data.
- **Open firmware** — the metering values are the iPDU's primary sensors; both
  the OpenBMC (Linux) and WallaBMC (Zephyr) tracks expose them via Redfish.

The MAXQ3180 datasheet and the reverse-engineered polling protocol are the
authority; the command catalog is documented with the model.
