# W83667HG-A — Super-I/O (KGPE-D16 host legacy I/O)

The Nuvoton (formerly Winbond) **W83667HG-A** is a QFP-128 LPC Super-I/O:
two 16550 UARTs, a PS/2 keyboard controller, ACPI power-sequencing glue, a
hardware monitor, watchdog, SPI-flash bridge, and ten GPIO banks behind a
single Extended-Function configuration interface.
[W83667HG-A DS p.1-2,33](#sources) On the ASUS KGPE-D16 it is board
reference **`OU1`**, a *host-side* chip hanging off the AMD SP5100's LPC bus
— none of the BMC's I2C buses reach it, but an open BMC firmware constantly
meets it, for four reasons:

- **LPC peer of the BMC.** The Super-I/O, the AST2050 BMC
  ({doc}`/hardware/soc-ast2050`) and the TPM header all sit as peripherals
  on the same SP5100 LPC bus (shared `LAD[3:0]`/`LFRAME#`/`SERIRQ`), so
  host I/O decode ranges must not collide with the BMC's KCS/LPC windows.
  [W83667HG-SUPERIO-WIRING.md §3](https://github.com/mithro/ai-shenanigans-for-bmcs/blob/main/asus-kgpe-d16-firmware/schematic-wiring/W83667HG-SUPERIO-WIRING.md)
- **The Serial-over-LAN console runs through it.** Its UART B is the host
  side of the 2:1 SOL mux (`QU8`, Pericom PI5C3257) whose other input is
  the BMC's UART1; `BMC_PRESENT#` selects which one owns the shared serial
  console.
  [AST2050-BMC-WIRING.md §12](https://github.com/mithro/ai-shenanigans-for-bmcs/blob/main/asus-kgpe-d16-firmware/schematic-wiring/AST2050-BMC-WIRING.md)
- **It owns much of the ATX power sequencing.** `PSON#` to the PSU, the
  `PWROK0-2` rails, ACPI sleep signals `SUSB#`(S3)/`SUSC#`(S5), `RSMRST#`
  and the `RSTOUT0-2#` resets are all Super-I/O pins — the platform states
  the BMC's power-control GPIOs observe and drive are physically
  implemented here.
  [W83667HG-SUPERIO-WIRING.md §6](https://github.com/mithro/ai-shenanigans-for-bmcs/blob/main/asus-kgpe-d16-firmware/schematic-wiring/W83667HG-SUPERIO-WIRING.md)
- **Its GPIO straps gate the NICs the BMC shares.** `GP37`/`GP50` drive
  `SIO_LAN1DISABLE#`/`SIO_LAN2DISABLE#` into the two Intel 82574L NICs
  ({doc}`intel-82574l`) — the same NICs that carry the BMC's NC-SI sideband
  — so a Super-I/O GPIO write can take down a BMC network path.
  [W83667HG-SUPERIO-WIRING.md §7](https://github.com/mithro/ai-shenanigans-for-bmcs/blob/main/asus-kgpe-d16-firmware/schematic-wiring/W83667HG-SUPERIO-WIRING.md)

## 1.2 Configuration access — Extended Function mode

All configuration lives behind an index/data pair: the **Extended Function
Index Register (EFIR)** at `0x2E` and **Extended Function Data Register
(EFDR)** at `0x2F` (or `0x4E`/`0x4F` when the HEFRAS strap — pin 31,
readable as CR26 bit 6 — is pulled high). Register writes are only accepted
in **Extended Function mode**, entered by two successive writes of `0x87`
to the EFER (same address as the EFIR) and left by writing `0xAA`.
[W83667HG-A DS p.33-36](#sources)

```text
; enter Extended Function mode
out 0x2E, 0x87            ; EFER <- 0x87 (first key write)
out 0x2E, 0x87            ; EFER <- 0x87 (second key write)

; select a logical device: global register CR07 = LDN
out 0x2E, 0x07            ; EFIR <- 0x07 (Logical Device Number register)
out 0x2F, 0x0B            ; EFDR <- LDN  (example: 0x0B = hardware monitor)

; read/write a configuration register inside that logical device
out 0x2E, 0x60            ; EFIR <- register index (CR60 = I/O base, high byte)
in  0x2F                  ; EFDR -> data (or out 0x2F, <data> to write)

; exit Extended Function mode
out 0x2E, 0xAA            ; EFER <- 0xAA
```

Global registers occupy indexes `0x00`–`0x2F` and are reachable without
selecting a logical device; indexes `0x30` and up address the currently
selected logical device (every device has `CR30` bit 0 as its
activate bit and `CR60`/`CR61` as its I/O base address, where
applicable). [W83667HG-A DS p.33-36](#sources)

### 1.5.3 ACPI (LDN A) — the power-sequencing brain

This logical device is where the board's power behaviour is configured;
several registers live in the `VBAT`/`VSB` wells so they survive main-power
loss. [W83667HG-A DS p.239-247](#sources)

```{list-table} ACPI logical-device key registers
:header-rows: 1
:widths: 12 14 74

* - Index
  - Default
  - Description
* - `E0`
  - `01` (VBAT)
  - Bit 7 DIS_PSIN blocks the panel switch from PSOUT#; bits 6/5 keyboard/mouse wake enables; PS/2 wake-key options
* - `E3`
  - `00`
  - Event status (read-clear): VSB power-loss (5), thermal shutdown (3), PSIN event (2), mouse/keyboard wake (1/0)
* - `E4`
  - `00`
  - Bits 6–5 power-loss policy after AC resume (G3→S5): `00` always off, `01` always on, `10` pre-state, `11` user-defined last-state; bit 4 enables 3VSBSW#
* - `E5`
  - `02`
  - GPIO reset sources; bit 1 PWROK source (0 = PSON#, 1 = SUSB#, default); bit 0 ATXPGD-controls-PWROK enable
* - `E6`
  - `1C`
  - Bit 6 SKTOCC status (VBAT); bit 5 CASEOPEN clear; bit 4 power-loss last-state flag (VBAT); bits 3–1 PWROK_DEL rise delay (200–860 ms taps); bit 0 PWROK_TRIG forces PWROK low
* - `E7`
  - `00`
  - Bit 4 EN_ONPSOUT (0.5 s PSOUT# pulse after AC-loss resume, for southbridges whose S3# idles low — AMD case); bit 3 WDTO# reset source; bit 0 hardware-monitor reset source (PWROK vs LRESET#)
* - `F2`
  - `7C` (VSB)
```

Source: [W83667HG-A DS p.239-247](#sources). The `PWROK0` output rises
300–600 ms (default tap) after `3VCC` is good, gated by `ATXPGD` from the

[W83667HG-A DS p.28,242](#sources)

### 1.5.4 Watchdog (LDN 8, WDTO#)

Off on this board (coreboot leaves LDN 8 inactive) but usable: `CRF6` is
the countdown value (`0x00` disables; 1–255 units), `CRF5` selects
second/minute units (bit 3) with a ×1000-faster test mode (bit 4) and can
route the timeout pulse to the `KBRST#` pin (bit 1), and `CRF7` carries
the timeout status (bit 4), a software trigger (bit 5) and the IRQ select.
A WDTO# event can optionally pulse `PWROK0/1/2` low for ~100 ms — a
hardware host reset. [W83667HG-A DS p.230-232](#sources)

### 1.5.5 Hardware monitor (LDN B) — the *secondary* monitor

`CR60`/`CR61` set the HM base address (2-byte boundary); the **Address
Port** and **Data Port** are then at base+5 and base+6 (coreboot sets base
`0x290` → ports `0x295`/`0x296`), behind which sit banked monitor
registers (bank select at index `0x4E`; indexes `0x00`–`0x4F` are global,
`0x50`–`0xFF` per-bank). `CR70` selects an IRQ — deliberately left
unassigned by coreboot "to prevent lockups". The block also hosts the
**SB-TSI** master (`CRE0`: bank/mode/SCL-frequency selects) that can read
AMD CPU temperature over the TSIC/TSID pins (113/115), and it is reachable
over I2C as a slave (address programmed at Bank 0 index `0x48`) as well as
over LPC — the datasheet warns the two must not access it simultaneously.
[W83667HG-A DS p.38-40,74,248](#sources)
[devicetree.cb:180](https://github.com/coreboot/coreboot/blob/4.11/src/mainboard/asus/kgpe-d16/devicetree.cb#L180)

It monitors nine voltages (`CPUVCORE`, `VBAT`, `3VSB`, `3VCC`, `AVCC`
internally; `VIN0-2` and `VCORE_REFIN` externally), five fan tachometers,
three temperatures (`SYSTIN`/`CPUTIN`/`AUXTIN`) and a case-open input via
an 8-bit ADC. [W83667HG-A DS p.14,38](#sources) On the KGPE-D16 only the
CPU-core sense pins (`CPUVCORE` pin 107, `VCORE_REFIN` pin 103) and
`CASEOPEN#` (pin 100, on the shared `INTRUDER#` net) are wired — the
fan/thermal monitoring duty belongs to the dedicated {doc}`w83795g`
(`QU4`) on the SP5100 SMBus, making the Super-I/O monitor a
CPU-voltage-only backup here.
[W83667HG-SUPERIO-WIRING.md §6-7](https://github.com/mithro/ai-shenanigans-for-bmcs/blob/main/asus-kgpe-d16-firmware/schematic-wiring/W83667HG-SUPERIO-WIRING.md)

### 1.6.2 The SOL serial path

UART A is a plain COM port: `SOUTA`/`SINA` (pins 34/33) plus full modem
control to RS-232 transceiver `U12` (AZ75232). UART B is the interesting
one: `SOUTB`/`SINB` (pins 72/71) and `RTSB`/`CTSB` (pins 69/67) run to the
**SOL mux `QU8`** (PI5C3257), whose other port carries the AST2050's UART1
(`TXD1`/`RXD1`/`NRTS1`/`NCTS1`); `BMC_PRESENT#` selects whether the host's
UART B or the BMC's UART1 owns the shared serial console. The remaining
UART B modem-control signals go to a second AZ75232 (`U13`).
[W83667HG-SUPERIO-WIRING.md §4](https://github.com/mithro/ai-shenanigans-for-bmcs/blob/main/asus-kgpe-d16-firmware/schematic-wiring/W83667HG-SUPERIO-WIRING.md)
[AST2050-BMC-WIRING.md §12](https://github.com/mithro/ai-shenanigans-for-bmcs/blob/main/asus-kgpe-d16-firmware/schematic-wiring/AST2050-BMC-WIRING.md)
Note the mux sits on the *board-level serial routing*: UART B itself is
just LDN 3 at `0x2F8` to the host.

### 1.6.4 LAN straps, PME and beep

- **Pin 76 `GP37`** drives `SIO_LAN1DISABLE#` (→ 82574L `LU1` + jumper
  `LAN_SW1`) and **pin 75 `GP50`** drives `SIO_LAN2DISABLE#` (→ `LU2` +
  `LAN_SW2`). These pins power up as GPIO (CR2C bits 7–6 = `00`); their
  events can be routed to PSOUT#/PME# via ACPI `CRFE`. Because the BMC's
  NC-SI sideband rides these NICs, the straps sit *upstream* of a BMC
  network path. [W83667HG-A DS p.210,246](#sources)
- **Pin 65 `PME#`** is bussed to every PCIe slot and both NICs
  (enable: ACPI `CRF2` bit 0). [W83667HG-A DS p.244](#sources)
- **Pin 51 `BEEP`** (default per CR27 bit 1) drives the chassis
  speaker/buzzer (`BUZZ1`, `PANEL1`). [W83667HG-A DS p.208](#sources)
- **Strap pins double as UART A modem-control**: HEFRAS is pin 31
  (`RTSA#`), the SPI-ROM enable strap is pin 32 (`DTRA#`), FANSET is pin
  34 (`SOUTA`), and the ASUS-function strap EN_AS is pin 62 — power-on
  pull resistors set them, then the pins revert to their runtime
  functions. [W83667HG-A DS p.25-26](#sources)

## 1.7 Linux and coreboot bindings

```{admonition} Driver support
:class: note

- **hwmon**: Linux [`w83627ehf`](https://github.com/torvalds/linux/blob/master/drivers/hwmon/w83627ehf.c)
  (`CONFIG_SENSORS_W83627EHF`). The mainline driver's supported list
  includes the W83667HG and W83667HG-B (both under the `w83667hg`
  prefix); it matches the Super-I/O ID with mask `0xFFF0`, so the
  W83667HG-A's `0xB35x` chip ID hits the `0xB350` entry and is detected
  and handled as the **W83667HG-B** variant (the original W83667HG is
  `0xA510`). [kernel.org w83627ehf.html](#sources) The NCT677x lineage
  continues from this part: the W83667HG-I *is* the Nuvoton NCT6775F,
  served by the separate
  [`nct6775`](https://github.com/torvalds/linux/blob/master/drivers/hwmon/nct6775-core.c)
  driver. [kernel.org w83627ehf.html](#sources)
- **watchdog**: Linux [`w83627hf_wdt`](https://github.com/torvalds/linux/blob/master/drivers/watchdog/w83627hf_wdt.c)
  (`CONFIG_W83627HF_WDT`) drives LDN 8's WDTO#; its chip list includes
  `w83667hg` (ID `0xA5`) and `w83667hg_b` (ID `0xB3`) — again the -A's
  `0xB3` high byte identifies it as the latter.
- **coreboot**: [`src/superio/winbond/w83667hg-a`](https://github.com/coreboot/coreboot/tree/4.11/src/superio/winbond/w83667hg-a)
  (coreboot 4.11, the last KGPE-D16 release). The KGPE-D16 mainboard
  selects `SUPERIO_WINBOND_W83667HG_A`
  ([Kconfig:15](https://github.com/coreboot/coreboot/blob/4.11/src/mainboard/asus/kgpe-d16/Kconfig#L15))
  and configures it at `2e` in
  [devicetree.cb:180](https://github.com/coreboot/coreboot/blob/4.11/src/mainboard/asus/kgpe-d16/devicetree.cb#L180):
  COM1/COM2, KBC, ACPI and the hardware monitor on; FDC, parallel, SPI,
  GPIO banks, watchdog, PECI and VID off.
```
