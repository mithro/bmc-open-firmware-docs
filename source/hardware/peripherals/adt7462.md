# ADT7462 — temperature/voltage monitor + fan controller

The **ADT7462** (originally Analog Devices, now onsemi; datasheet ADT7462/D Rev 6)
is a "flexible" system monitor: up to **4 temperature channels** (1 local + 3
remote diodes), up to **13 voltage inputs**, **8 tachometer** inputs, and **4 PWM**
fan-drive outputs with a full automatic fan-control engine. Because it has more
functions than pins, most inputs are **pin-strap multiplexed** (a given register
means either a voltage or a tach/temperature depending on the pin configuration —
see the multiplexing note below). [ADT7462 DS p.1,10-14,22,31](#sources)

The C410X uses **two ADT7462s** on bus `0xF1` (`&i2c1`) behind a PCA9544A 4-channel
mux at `0x70`: chip #1 at 7-bit `0x58` (ADD pin strapped low; 8-bit `0xB0`),
chip #2 at `0x5C` (ADD pin floating; 8-bit `0xB8`). (`0xB0`/`0xB8` are the 8-bit
device addresses; the exact PCA9544A channel each chip sits on is not pinned down
in the RE notes — see {doc}`pca954x-mux`.)
Together they provide 6 board-temperature zones (IPMI sensors `0x01`–`0x06`) and
tach/PWM for all 8 chassis fans (IPMI fan sensors `0x80`–`0x87`). Firmware drivers:
`G_sOEMADT7462_I2CTEMP_IOSAPI` (temp), `_I2CFAN_IOSAPI` (fans), `_I2CADC_IOSAPI`
(voltage). [IS_fl.bin.md:81-95,141-163](https://github.com/mithro/ai-shenanigans-for-bmcs/blob/main/dell-c410x-firmware/io-tables/IS_fl.bin.md#L81-L95) [dts:596-663] [ANALYSIS.md:496-498](https://github.com/mithro/ai-shenanigans-for-bmcs/blob/main/dell-c410x-firmware/ANALYSIS.md#L496-L498)
[ADT7462 DS p.15](#sources)

```{admonition} C410X firmware "Local Temp" register caveat
:class: warning

The firmware reads Board Temp 1/2/3 from ADT7462 registers `0x8B`, `0x8D`, `0x8F`
and labels `0x8F` "Local Temp". In the datasheet those are the **Remote 1, Remote 2
and Remote 3** temperature MSB registers respectively — the true on-die **Local**
channel MSB is `0x89`, which the firmware does not read. So the C410X actually uses
the three *remote* diode channels and merely names the third one "Local".
[IS_fl.bin.md:88-95](https://github.com/mithro/ai-shenanigans-for-bmcs/blob/main/dell-c410x-firmware/io-tables/IS_fl.bin.md#L88-L95) [ADT7462 DS p.19](#sources)
```

## Temperature channels & conversion

Four channels: Local (`0x89` MSB / `0x88` LSB), Remote 1 (`0x8B`/`0x8A`), Remote 2
(`0x8D`/`0x8C`), Remote 3 (`0x8F`/`0x8E`). Data is **offset binary with a −64 °C
offset**; the MSB register is 1 °C/LSB and the LSB register's bits [7:6] add 0.25 °C
resolution (10-bit total). Range −64 °C to +191 °C. [ADT7462 DS p.19-20](#sources)

$$
\begin{aligned}
\text{Temp}(\text{°C}) &= \text{MSB} - 64 && (1\,\text{°C})\\
\text{Temp}(\text{°C}) &= \frac{(\text{MSB} \ll 2)\ \mathbin{|}\ \text{LSB}[7{:}6]}{4} - 64 && (0.25\,\text{°C})
\end{aligned}
$$

Checks from the datasheet: −64 °C→`0x00`, 0 °C→`0x40`, +25 °C→`0x59`, +100 °C→`0xA4`.
**Read the LSB register first** — this freezes the MSB against a torn read.
[ADT7462 DS p.20](#sources) Temperature offset registers `0x56`–`0x59` (two's-complement,
0.5 °C/LSB) trim each channel. Limits: low `0x44`–`0x47`, high `0x48`–`0x4B`,
THERM1 `0x4C`–`0x4F`, THERM2 `0x50`–`0x53`. [ADT7462 DS p.21](#sources)

## Voltage channels & conversion

A 10-bit successive-approximation ADC with a 0–2.25 V core range and on-chip
attenuators for higher rails; the nominal rail sits at **3/4 of full scale**
(768 = `0x300`). Up to 13 inputs, several sharing registers with temp/tach pins.
[ADT7462 DS p.22](#sources)

$$
\text{Voltage} = \text{Code} \times 1\,\text{LSB}
$$

(per-rail LSB, e.g. +12V: 0.0625 V, 16 V full-scale.)
Representative LSBs: +12V rails 0.0625 V (FS 16 V); +5V 0.026 V (FS 6.67 V); +3.3V
0.0172 V (FS 4.4 V); +1.5V 0.0078 V (FS 2.0 V). Example: 1.8 V on a 1.8 V channel →
$1.8 / 0.0094 = 192 = \tfrac{3}{4}$ scale. Attenuators are bypassable per pin via `0x18`/`0x19`
(then keep the pin ≤ 2.25 V). [ADT7462 DS p.22-25](#sources)

## Tachometer / fan-speed & RPM conversion

Eight tach channels; each is a 16-bit count (LSB + MSB registers). The C410X reads
TACH1 `0x98`, TACH2 `0x9A`, TACH3 `0x9C`, TACH4 `0x9E` — matching the datasheet
TACH1–4 LSB registers (the firmware TACH↔fan wiring is non-sequential).
[IS_fl.bin.md:146-159](https://github.com/mithro/ai-shenanigans-for-bmcs/blob/main/dell-c410x-firmware/io-tables/IS_fl.bin.md#L146-L159) [ADT7462 DS p.31](#sources)

$$
\text{Fan Speed (RPM)} = \frac{f \times 60}{\text{TachCount}}
$$

with $f = 90\,\text{kHz}$ (tach clock).
`TachCount` = 16-bit MSB:LSB value; the count is 90 kHz clocks gated over the tach
period (2 pulses/rev assumed; divide by 1.5 for 6-pole, by 2 for 8-pole fans).
Example: `0x17FF` = 6143 → $\dfrac{90000 \times 60}{6143} \approx 879\,\text{RPM}$. `0xFFFF` = stalled /
< 100 RPM (also the reset value). **Read the LSB first** to freeze both bytes.
Enable per-tach measurement in `0x07`. [ADT7462 DS p.31-32](#sources)

## PWM outputs & automatic fan control

register (`0x21`–`0x24`) selects its behaviour:

```{list-table} PWM config register (0x21–0x24) fields
* - 4
  - INV
  - 0 = PWM active-low; 1 = active-high (default). [ADT7462 DS p.70](#sources)
* - 7:5
  - BHVR (control source)
  - 000=Local, 001=Remote1, 010=Remote2, 011=Remote3, 100=Off, 101=max(Local,Remote3), 110=max(all), 111=Manual. [ADT7462 DS p.70](#sources)
```

**Fan-control modes.** `BHVR = 111` is **manual** — software writes the duty in
`0xAA`–`0xAD` directly (this is how a host fan-control daemon drives the fans). Any
other BHVR value is **automatic**: the PWM duty is a linear ramp of the selected
temperature channel: [ADT7462 DS p.33-39](#sources)

- **TMIN** (`0x5C`–`0x5F`, default 90 °C): the temperature at which the fan starts.
  Below TMIN − THYST the fan turns off (or holds PWMMIN if the per-PWM Min bit is
  set). [ADT7462 DS p.35](#sources)
- **PWMMIN** (`0x28`–`0x2B`, default 50 %): the duty at TMIN.
- **TRANGE** (`0x60`–`0x63` [7:4], default 32 °C): the temperature span over which
  duty ramps from PWMMIN toward 100 % — i.e. the ramp *slope*. Encoded 0000 = 2 °C
  … 1100 = 32 °C (default) … 1111 = 80 °C. [ADT7462 DS p.76](#sources)
- **PWMMAX** (`0x2C`, default 75 %): the ceiling before THERM forces full speed.
- **Fan THYST** (`0x60`–`0x63` [3:0], default 4 °C): on/off hysteresis.

The fan reaches full speed at $\text{TMAX} = \text{TMIN} + (255 - \text{PWMMIN}) \times \text{TRANGE} / 170$ (which
simplifies to $\text{TMIN} + \text{TRANGE}$ when PWMMIN = 33 %); duty is linear in temperature
between TMIN and TMAX. Exceeding a **THERM limit** (`0x4C`–`0x53`) forces all fans
to 100 % and cannot be masked. Spin-up runs 100 % until 2 tach pulses are seen.
[ADT7462 DS p.38-40](#sources) The device also supports a **dynamic-TMIN** loop (`0x0B`/`0x0C`,
operating-point `0x5A`/`0x5B`) that self-adjusts TMIN to hold each zone near a
target, and acoustic ramp-rate smoothing (`0x1A`/`0x1B`). [ADT7462 DS p.43-48](#sources)

## Interrupt / status & THERM

Interrupt sources latch into the **Host** status registers `0xB8` (thermal 1),
`0xB9` (thermal 2 + VRD), `0xBA` (per-channel THERM), `0xBB`/`0xBC` (voltage 1/2),
`0xBD` (fan), `0xBE` (digital incl. chassis-intrusion bit 7), `0xBF` (GPIO), with
BMC-side duplicates at `0xC0`–`0xC6`. Each source can be masked via `0x30`–`0x38`.
Status bits are sticky and clear on read once the condition clears; the CI latch
persists on VBATT even when powered off. In **SMBALERT** mode the device answers
the Alert Response Address `0x0C`; in comparator mode ALERT tracks the condition.
[ADT7462 DS p.15,18,27,71-73,78-80](#sources)

## Pin-strap multiplexing (why registers alias)

Because there are more functions than pins, the firmware picks an Easy Config
option (`0x14`) and trims Pin Config `0x10`–`0x13`, after which several value/limit
registers alias between a voltage and a tach/temperature meaning: `0x8B` = Remote 1
temp **or** pin-15 voltage; `0x8F` = Remote 3 temp **or** pin-19 voltage;
`0xA3`/`0xA5`/`0xA7`/`0xA9` = TACH5–8 **or** +12V1/+12V2/+5V/+12V3 voltage; `0x78`
= TACH1 limit **or** VID limit. Remote 2 (`0x8C`/`0x8D`) has no voltage alternate,
so it is temperature-only. A model or driver must read Pin Config `0x10`–`0x13` to
know which meaning is active. [ADT7462 DS p.10-14,24,31,65-66](#sources)

## Linux binding ([`adt7462`](https://github.com/torvalds/linux/blob/master/drivers/hwmon/adt7462.c))

Mainline Linux drives this with `hwmon`/[`adt7462`](https://github.com/torvalds/linux/blob/master/drivers/hwmon/adt7462.c). The C410X device tree uses
`compatible = "adi,adt7462"`; the current mainline driver also matches the onsemi
vendor prefix (`onnn,adt7462`) and the plain I2C device id [`adt7462`](https://github.com/torvalds/linux/blob/master/drivers/hwmon/adt7462.c). [dts:625,648]
[adt7462.c](https://github.com/torvalds/linux/blob/master/drivers/hwmon/adt7462.c) The driver probes via Device/Company/Revision IDs (`0x3D`=`0x62`,
`0x3E`=`0x41`, `0x3F`=`0x04`) and its register `#define`s match the datasheet
(temp base `0x88`, min/max temp `0x44`/`0x48`, fan/tach `0x98`–`0x9F` + `0xA2`–`0xA9`,
fan enable `0x07`, fan-min `0x78`–`0x7F`, PWM `0xAA`–`0xAD`, PWMMIN `0x28`–`0x2B`,
PWMMAX `0x2C`, TMIN `0x5C`–`0x5F`, TRANGE `0x60`–`0x63`, pin-config `0x10`–`0x13`).
It models 4 temps, 13 voltages and 8 fans, decoding the pin-strap array to resolve
the register aliasing, and reads only the Host status registers. It exposes hwmon
attributes `tempN_input/min/max/label/alarm` and `tempN_auto_point{1,2}_temp/hyst`,
`inN_input/min/max/label/alarm`, `fanN_input/min/alarm`, and
`pwmN`/`pwmN_enable`/`pwmN_auto_point{1,2}_pwm`/`pwmN_auto_channels_temp` plus
`force_pwm_max`; the `auto_point` PWM/temperature attributes map to
PWMMIN/PWMMAX and TMIN/TMIN+TRANGE. [adt7462.c](https://github.com/torvalds/linux/blob/master/drivers/hwmon/adt7462.c)
