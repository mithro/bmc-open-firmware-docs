# ADT7462 — thermal monitor + fan controller

An Analog Devices monitor combining multi-channel temperature and voltage
sensing with **4 channels of PWM fan control** and tachometer inputs. The C410X
uses **two** (bus `0xF1`, behind the PCA9544 mux at addresses `0x58` and `0x5C`)
to drive and monitor its 8 fans and read board temperatures/voltages.

```{list-table}
:header-rows: 0
:widths: 30 70

* - Interface
  - I2C, 8-bit register pointer, 8-bit data
* - Addresses
  - `0x58`, `0x5C` (bus 0xF1 via PCA9544)
* - Channels
  - temperature, voltage, 4× PWM fan out, 4× tach in
```

## Model / driver notes

- Register-pointer protocol (write pointer, then read/write data byte).
- Key registers for a model: device/revision ID, temperature readings, fan tach
  counts, PWM duty registers, and configuration/limit registers.
- Mainline Linux binds `hwmon`/`adt7462`. OpenBMC fan control (`phosphor-pid-control`)
  drives PWM from the temperature inputs; the model must let a written PWM value
  read back and produce a plausible tach reading so control loops converge.

Detailed bit tables are added with the model; the ADT7462 datasheet is the
authority.
