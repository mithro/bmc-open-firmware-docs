# ICS1893AFLF — 10/100 Ethernet PHY

The ICS1893AFLF (marked "P47932M / 1893AFLF") is a 3.3 V 10Base-T/100Base-TX
integrated "PHYceiver" from ICS (now Renesas/IDT). It bonds the NS9360's
on-chip 10/100 MAC to the RJ-45 management port; the 25 MHz crystal Y2 is its
reference clock. `[ANALYSIS.md](https://github.com/mithro/ai-shenanigans-for-bmcs/blob/main/hpe-ipdu-firmware/ANALYSIS.md)` `[STATUS.md](https://github.com/mithro/ai-shenanigans-for-bmcs/blob/main/hpe-ipdu-firmware/STATUS.md)`

## MAC ↔ PHY connection (NS9360 side)

two-wire SMI management bus (MDIO/MDC). The relevant NS9360 GPIO mux
assignments: `[ANALYSIS.md](https://github.com/mithro/ai-shenanigans-for-bmcs/blob/main/hpe-ipdu-firmware/ANALYSIS.md)`

The NS9360 also needs a separate MDC (management clock) output; MDIO/MDC form
the clause-22 Serial Management Interface. The MAC transports management frames
of the form `PRE(32) ST(01) OP RD/WR PHYAD(5) REGAD(5) TA(2) DATA(16)`.
`[ICS1893 DS p.60](#sources)` (Table 7-2, Management Frame Structure);
cross-checked against `[Linux mii.h](#sources)`.

## PHY address strapping (LED/status pins)

The 5-bit PHY (SMI) address is **latched at reset from the five multiplexed
status/LED pins** P0AC, P1CL, P2LI, P3TD, P4RD, and thereafter appears in
Extended Control Register bits 16.10:6. `[ICS1893 DS p.88-89](#sources)`

- Each LED pin is sampled through a 10 kΩ resistor at reset; pulling a pin high
  (or low) sets the corresponding address bit. `[ICS1893 DS p.106](#sources)` (§9.3.2)
- **PHY address 00000 tri-states the MII interface** and must not be used for a
  live port. `[ICS1893 DS p.60](#sources)` Consequently the iPDU straps a non-zero
  address; a non-zero address also forces the Isolate bit (0.10) default to 0.
  `[ICS1893 DS p.65](#sources)`

ICS vendor registers 16–19; registers 9–15 and 20–31 are reserved. Register

## Registers 2/3 — PHY Identifier (PHYSID1/2) `[ICS1893 DS p.72-75](#sources)`

- OUI (Integrated Circuit Systems) = **0x00A0BE**. `[ICS1893 DS p.73](#sources)` (Table 8-8)
- Register 2 (PHYSID1) default = **0x0015** (OUI bits 3–18).
- Register 3 (PHYSID2) default = **0xF441** = OUI bits 19–24 (`111101`), model
  number 4 (`000100`, bits 3.9:4), revision 1 (`0001`, bits 3.3:0).
  `[ICS1893 DS p.74-75](#sources)` (Tables 8-9, 8-10)
- Registers 2/3 map to Linux `MII_PHYSID1`/`MII_PHYSID2` and yield a 32-bit
  PHY ID of `0x0015F441` (masking the revision nibble is customary). `[Linux mii.h](#sources)`

## Register 5 — Link Partner Ability (ANLPAR) `[ICS1893 DS p.80](#sources)`

Read-only image of the partner's base Link Code Word (valid only once AN
completes, i.e. bit 1.5/17.4 = 1). Bit layout mirrors register 4: 5.15 Next
Page, 5.14 Acknowledge, 5.13 Remote fault, 5.12:5 Technology Ability Field
(100TX FD/HD, 10T FD/HD in bits 5.8:5), 5.4:0 Selector field. Reset 0x0000.
Maps to Linux `MII_LPA`. `[ICS1893 DS p.80-81](#sources)` `[Linux mii.h](#sources)`

## Registers 7/8 — Auto-Negotiation Next Page `[ICS1893 DS p.84-87](#sources)`

- Register 7 (Next Page **Transmit**, R/W, reset **0x2001**): 7.15 Next Page,
  7.13 Message Page (default 1), 7.12 Acknowledge 2, 7.11 Toggle (RO), 7.10:0
  Message/Unformatted Code field.
- Register 8 (Next Page **Link Partner Ability**, RO, reset 0x0000): same bit
  layout, populated from the partner's next-page word.
