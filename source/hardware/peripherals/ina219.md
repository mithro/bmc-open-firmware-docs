# INA219 — current/power monitor

A Texas Instruments high-side current/voltage/power monitor with an I2C
interface. On the Dell C410X, **16 of them** sit on bus `0xF0` (addresses
`0x40`–`0x4F`), one per PCIe slot, measuring GPU power draw.

```{list-table}
:header-rows: 0
:widths: 30 70

* - Interface
  - I2C, up to 2.56 MHz; 16-bit big-endian registers
* - Address
  - `0x40`–`0x4F` (A1/A0 strap; 16 slots use all 16 addresses)
* - Register pointer
  - set by a write; auto-retained for repeated reads
```

## Register map

```{list-table}
:header-rows: 1
:widths: 10 22 12 12 44

* - Addr
  - Register
  - Access
  - Reset
  - Notes
* - `0x00`
  - Configuration
  - R/W
  - `0x399F`
  - range, gain, ADC resolution/averaging, mode; bit15 = reset
* - `0x01`
  - Shunt voltage
  - R
  - `0x0000`
  - signed; LSB = 10 µV
* - `0x02`
  - Bus voltage
  - R
  - `0x0000`
  - bits[15:3] = voltage (LSB 4 mV); bit1 CNVR; bit0 OVF
* - `0x03`
  - Power
  - R
  - `0x0000`
  - valid only after Calibration is written
* - `0x04`
  - Current
  - R
  - `0x0000`
  - signed; derived from shunt × calibration
* - `0x05`
  - Calibration
  - R/W
  - `0x0000`
  - sets current/power LSB; gates Power/Current registers
```

## Behaviour a model must implement

- **Pointer register**: an I2C write of one byte sets the register pointer; a
  write of three bytes sets pointer + 16-bit value; a read returns the pointed
  register (MSB first) and does *not* advance the pointer.
- **Reset defaults**: Configuration reads back `0x399F` before any write;
  Calibration reads `0x0000`.
- **Config bit15 (reset)**: writing 1 restores all registers to reset defaults.
- **Measurement registers** may be modelled as programmable/static values (a
  test seeds shunt/bus voltage and calibration, then reads current/power). A
  faithful model computes Power/Current from shunt × calibration once
  Calibration is non-zero.

## Driver

Mainline Linux binds this as `hwmon`/`ina2xx` (`ti,ina219`). No custom driver is
needed; the device tree lists 16 nodes on the `0xF0` bus. The OpenBMC
`dbus-sensors` `ADCSensor`/`IntrusionSensor` equivalents read them via hwmon.

## Test bench

`tests/qtest/ina219-test.c` asserts: reset value of Configuration/Calibration,
pointer-write then read-back, config reset bit, and that a seeded
shunt+calibration yields the expected Current/Power. The bench **fails** if any
register returns the unimplemented-default `0x0000` where a reset value is
specified. See {doc}`../../emulation/testbench`.
