# INA219 — current / voltage / power monitor

The Texas Instruments **INA219** (datasheet SBOS448G) is a high-side
current-shunt and bus-voltage monitor with an internal 12-bit ADC, a programmable
calibration multiplier, and an I2C/SMBus interface. It measures the voltage drop
across an external shunt resistor (to derive current) and the bus supply voltage,
and computes power on-chip. [INA219 DS p.13] The C410X uses **16 of them**, one
per PCIe slot, on the dedicated I2C bus `0xF0` (`&i2c0`), sitting on each slot's
12 V rail with an (estimated) 2 mΩ shunt to measure GPU power draw. IPMI sensors
`0x50`–`0x5F` ("PCIE n Watt") read register `0x04` from each chip.
[IS_fl.bin.md:165-195] [dts:402-413]

The bus is isolated behind a buffer controlled by GPIOE3 (`ina219-bus-en`),
enabled only after PWRGD confirms the 12 V rail is stable, so that unpowered
INA219s cannot hang the bus during standby. [dts:336-341,406-408]

## Register map

All six INA219 registers are 16-bit and are transferred as two 8-bit bytes,
**most-significant byte first**. Register contents update 4 µs after a write, so a
4 µs delay is needed before reading back at SCL > 1 MHz. [INA219 DS p.18]

```{list-table} INA219 register set (16-bit registers, pointer-addressed)
:header-rows: 1
:widths: 10 22 10 12 46

* - Pointer
  - Register
  - Access
  - Reset
  - Function
* - `0x00`
  - Configuration
  - RW
  - `0x399F`
  - All-register reset (bit 15), bus-voltage range, PGA gain, bus/shunt ADC resolution & averaging, operating mode. [INA219 DS p.18-20]
* - `0x01`
  - Shunt Voltage
  - RO
  - measured
  - Signed shunt-voltage measurement, LSB = 10 µV; sign-extension depends on PGA. [INA219 DS p.20-22]
* - `0x02`
  - Bus Voltage
  - RO
  - measured
  - Bus voltage in bits [15:3] (LSB = 4 mV); bit 1 = CNVR (conversion ready), bit 0 = OVF (math overflow). [INA219 DS p.23]
* - `0x03`
  - Power
  - RO
  - `0x0000`
  - Power = Current × Bus Voltage; LSB = 20 × Current_LSB. Reads 0 until Calibration is programmed. [INA219 DS p.18,23]
* - `0x04`
  - Current
  - RO
  - `0x0000`

  - RW
  - `0x0000`
```

The IPMI sensor table reads register `0x04` for each slot (labelled "Power" in the
firmware notes, though `0x04` is the Current register in the datasheet; the
firmware IOSAPI `G_sOEMINA219_I2CADC` applies the shunt/calibration scaling to
convert to watts). [IS_fl.bin.md:71-72,190-192] [ANALYSIS.md:494]

## Data registers, formats & conversions

**Shunt Voltage (0x01):** LSB = 10 µV, two's-complement, sign bit extended left
according to the PGA range. Full-scale magnitude tracks the PGA setting: ÷8 =
±320 mV (±32000 counts), ÷4 = ±160 mV (±16000), ÷2 = ±80 mV (±8000), ÷1 = ±40 mV
(±4000). The number of significant data bits (and how many high bits are sign
copies) therefore changes with PGA. [INA219 DS p.20-22]

**Bus Voltage (0x02):** data is in bits [15:3]; bit 2 is reserved. To convert:
take the register value, shift right 3, multiply by the 4 mV LSB. Full-scale =
32 V (8000 counts, `0x1F40`) or 16 V (4000 counts, `0x0FA0`) per BRNG.
[INA219 DS p.23]

- **CNVR (bit 1):** Conversion Ready — set when a new conversion result is
  available; cleared by reading the Power register or by writing a new operating
  mode (except power-down/disable). [INA219 DS p.23]
- **OVF (bit 0):** Math Overflow — set when the Power/Current calculation overflows
  (result may be meaningless). [INA219 DS p.23]

**Power (0x03) / Current (0x04):** both are computed on-chip and read as 0 until
the Calibration register is non-zero. Current is signed two's-complement.
[INA219 DS p.18,23]

The measurement engine math (all verified against the datasheet worked example,
[INA219 DS p.25-26]):

```
Current_LSB  = Max_Expected_Current / 32768          (choose a round value, e.g. 1 mA/bit)
Cal (0x05)   = trunc( 0.04096 / (Current_LSB × R_shunt) )     [Eq. 1]
Current(0x04)= (ShuntVoltage_reg × Cal) / 4096                [Eq. 4]
Power(0x03)  = (Current_reg × BusVoltage_shifted) / 5000      [Eq. 5]
Power_LSB    = 20 × Current_LSB
Corrected_Cal= trunc( Cal × MeasuredShuntCurrent / INA219_Current )   [Eq. 6, trim to a measured reference]
```
[INA219 DS p.13,23-24]

Worked example from the datasheet (max load 15 A, nominal 10 A, V_CM = 12 V,
R_shunt = 2 mΩ, PGA ÷1 = ±40 mV FSR, BRNG = 16 V): Current_LSB rounded to 1 mA/bit,
Cal = 20480 (`0x5000`). Then Shunt = `0x07D0` (2000 × 10 µV = 20 mV), Bus =
`0x5D98` (raw), shifted → 2995 × 4 mV = 11.98 V, Current = `0x2710` = 10000 ×
1 mA = 10.0 A, Power = `0x1766` = 5990 × 20 mW = 119.8 W. [INA219 DS p.25-26]

**Calibration (0x05):** bits FS15:FS1 hold the value; **FS0 is a void bit and
always reads 0** (cannot be written 1). Suitable for system-level trim via
Equation 6. [INA219 DS p.24]

## I2C addressing — how 16 instances coexist

Each INA219 has two address pins, **A0** and **A1**, each of which may be tied to
one of four references (GND, V_S+, SDA, SCL), giving **16 unique 7-bit addresses
`0x40`–`0x4F`** (fixed prefix `100`, low 4 bits from the pin combination). This is
exactly what lets the 16 C410X monitors share one bus with no multiplexer.
[INA219 DS p.14] [IS_fl.bin.md:194-195] [dts:410-411]

**Register-pointer protocol:** a write frame is `slave-addr(W)` → pointer byte →
[MSB, LSB] data. A read frame is `slave-addr(R)` → [MSB, LSB]; the pointer is
**retained** across reads, so repeated reads of the same register need no new
pointer write. The INA219 has a 28 ms SMBus timeout and supports F/S (≤ 400 kHz)
and HS (≤ 2.56 MHz) modes. A software reset is available via the RST bit or the
I2C General Call Reset. [INA219 DS p.14-15,17]

## Linux binding (`ina2xx`)

Mainline Linux drives this with the `hwmon`/`ina2xx` driver; the device tree uses
`compatible = "ti,ina219"` with a `shunt-resistor` property in **micro-ohms**
(driver default `INA2XX_RSHUNT_DEFAULT` = 10000 µΩ = 10 mΩ). The C410X DT declares
all 16 nodes on `&i2c0` with `shunt-resistor = <2000>` (2 mΩ, estimated pending
board measurement). [dts:422-427] [ina2xx.c] [ti,ina2xx.yaml]

The driver's register `#define`s (`INA2XX_CONFIG` 0x00 … `INA2XX_CALIBRATION`
0x05) and `INA219_CONFIG_DEFAULT = 0x399F` match the datasheet exactly.
Rather than solving Equation 1, the driver writes a **fixed** Calibration value of
4096 (`calibration_value`), which makes Equation 4 collapse to
Current_reg = Shunt_reg, and then does the amps/watts scaling in software from
`shunt-resistor` (using `shunt_div = 100` → 10 µV LSB, `bus_voltage_lsb = 4000`
µV with `bus_voltage_shift = 3`, and `power_lsb_factor = 20`). [ina2xx.c] It
exposes hwmon attributes `in0_input` (shunt voltage, mV), `in1_input` (bus
voltage, mV), `curr1_input` (mA) and `power1_input` (µW); the binding also allows
`ti,shunt-gain` (PGA divisor 1/2/4/8) and `ti,alert-polarity-active-high`. No
board-specific driver is required. [ina2xx.c] [ti,ina2xx.yaml]
