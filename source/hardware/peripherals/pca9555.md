# PCA9555 — 16-bit I2C GPIO expander

A 16-bit I2C GPIO expander. The C410X uses **five** of them for slot presence
detect, per-slot power-good feedback, attention buttons + slot power control, MRL
(manual retention latch) sensing, and PSU management / fan-status LEDs. Together
they contribute 80 of the board's 118 GPIO pins.

```{list-table}
:header-rows: 0
:widths: 30 70

* - Interface
  - I2C, 8-bit register pointer
* - Addresses
  - `0x20`–`0x23` (bus 0xF6) and `0x20` (bus 0xF1)
* - Registers
  - `0x00/0x01` input, `0x02/0x03` output, `0x04/0x05` polarity inv,
    `0x06/0x07` config (direction)
```

## Model / driver notes

- Config register bits select input (1) vs. output (0) per pin; reset = all
  inputs (`0xFF`).
- Output register drives pins configured as outputs; input register reflects pin
  state (a model exposes settable input lines so a test can assert presence /
  power-good / button events).
- Mainline Linux binds `gpio`/`pca953x` (`nxp,pca9555`). The device tree names
  each line and uses GPIO *hogs* for the fixed-function outputs that implement
  the 12-step power-on sequence.
- For the C410X `c410x-board-bench`, the model's input lines are what let the
  power-sequencing test observe the correct ordered GPIO transitions.
