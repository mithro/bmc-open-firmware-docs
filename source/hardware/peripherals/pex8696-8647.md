# PLX PEX8696 / PEX8647 — PCIe switches

The Dell C410X routes its 16 GPU slots through **4× PLX PEX8696** (96-lane,
24-port) downstream switches and **2× PLX PEX8647** (48-lane, 3-port) upstream
switches. The BMC controls them over an **I2C management interface** (bus `0xF3`)
to sequence slot power, handle hot-plug, and reconfigure host-to-slot routing
(multi-host modes such as 2:1, 4:1, 8:1).

```{list-table}
:header-rows: 0
:widths: 30 70

* - Interface
  - I2C/SMBus management port (not the PCIe data path)
* - PEX8696 addresses
  - `0x30`, `0x32`, `0x34`, `0x36` (bus 0xF3)
* - PEX8647 addresses
  - `0xD0`, `0xD4` (bus 0xF3)
```

## I2C command format

The PLX parts do **not** use a simple register-pointer protocol; they use a
4-byte command framing that carries a command code, a per-byte enable mask, and
a DWORD register index:

```{list-table}
:header-rows: 1
:widths: 18 20 62

* - Command
  - Code
  - Meaning
* - I2C write
  - `0x03`
  - write 4 data bytes to the addressed config DWORD
* - I2C read
  - `0x04`
  - read 4 data bytes from the addressed config DWORD
```

- The **byte-enable mask** (typically all four bytes) selects which bytes of the
  DWORD are affected.
- The **register DWORD index** is encoded in the command bytes; it selects a
  station/port config register.

## Operations the firmware performs

Reverse engineering of the stock firmware recovered the switch-management
routines: per-slot power on/off, all-slots-off, hot-plug enable, multi-host
routing configuration, and serial-EEPROM read/write. These are the operations
the OpenBMC PCIe-switch daemon re-implements (see {doc}`../../firmware/openbmc`).

## Model scope

A full PEX model is large; the QEMU model scopes to the **register set the
firmware actually touches** — the management command framing, the config DWORDs
read/written during power/hot-plug/multi-host sequences, and the EEPROM shadow.
The qtest bench replays recorded command sequences and checks the resulting
register state. This is the hardest peripheral model in the program and lands
after the simpler sensors.
